rEFInd Boot Manager (Open-Source) Installation Guide

rEFInd is a powerful and flexible UEFI boot manager that automatically detects multiple operating systems. It’s open-source, free, and a solid alternative to tools like BootIt Bare Metal.

Why Use rEFInd?

  • Automatic OS Detection: Scans for boot loaders on each UEFI-capable drive.
  • Multiple OS Support: Works with Linux (including Pop!_OS), Windows, macOS, and more.
  • Easy Installation: Installs from within your existing Linux OS — no need to burn a separate ISO.
  • Theming & Customization: Customize menu icons, colors, and layout to your liking.

Installation Overview (No Separate ISO Needed)

Below are the basic steps to install rEFInd directly from a Linux distribution (e.g., Pop!_OS, Ubuntu, Debian, etc.). You do not have to burn an ISO or boot from external media if you already have a functioning Linux environment.

  1. Download rEFInd
    Visit the official rEFInd website: https://www.rodsbooks.com/refind/.
    Download the latest ZIP archive or find a package for your distribution.
  2. Extract and Install
    If you downloaded the ZIP file, extract it, then open a terminal and navigate into the extracted folder. Run the installation script: 
    sudo ./refind-install
    This copies rEFInd files to your EFI partition and configures your UEFI firmware to recognize rEFInd.

    Tip: Some distros provide a package. If you’re on Ubuntu/Pop!_OS, you might do: 

    sudo apt update
sudo apt install refind

  3. Reboot
    Once installed, reboot your machine. You should see a new graphical boot menu listing all detected OSes.

Troubleshooting

  • Secure Boot Issues:
    If Secure Boot is enabled, you may need to disable it or sign rEFInd’s binaries. Check your BIOS/UEFI settings.
  • Missing EFI Partitions:
    Ensure each drive with an operating system has a valid EFI partition and that your firmware can detect it.
  • Manual Entries:
    In rare cases, you may need to edit refind.conf in the /boot/efi/EFI/refind/ directory to add custom boot stanzas if auto-detection doesn’t find certain OS installs.

Other Open-Source Boot Managers

  • GRUB (GRand Unified Bootloader):
    Default for many Linux distros. You can run sudo update-grub to detect other OSes.
  • systemd-boot (formerly gummiboot):
    A minimalist UEFI boot manager often found on distributions like Pop!_OS. Configuration requires editing loader entries in /boot/efi/loader/entries/.

Enjoy your new boot manager!
With rEFInd installed, you’ll be able to easily switch between different OSes installed on multiple NVMe or SATA drives without burning any new ISO images.
